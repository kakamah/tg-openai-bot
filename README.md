## Overview

Decoupled Telegram chatbot where the Aiogram bot and Django backend run as independent services. The bot communicates with the backend exclusively over HTTP — it never touches the database directly. Users authenticate via Telegram, conversations are persisted in SQLite, and responses are generated by GPT-5.5 with weather function calling support.

---

## Architecture

```
Telegram User
      │
      ▼
  Aiogram Bot (bot.py)          ← Independent, async, FSM-based
      │
      │  HTTP (aiohttp) + Token auth
      ▼
  Django REST API                ← Independent, DRF, Token Authentication
      │
      ├── SQLite (messages & users)
      ├── OpenAI GPT-5.5 (chat + function calling)
      └── WeatherAPI (tool call)
```

## Setup

```bash
# 1. Clone & enter project
git clone https://github.com/YOUR_USERNAME/your-repo.git
cd your-repo

# 2. Virtual environment
python -m venv venv && source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Environment variables
cp .env.example .env
```

`.env` contents:

```env
BOT_TOKEN=your-telegram-bot-token
OPENAI_API_KEY=sk-...
WEATHER_API_KEY=your-weatherapi-key
API_BASE=http://127.0.0.1:8000/api
```

```bash
# 5. Migrate & run Django
python manage.py migrate
python manage.py runserver

# 6. Run bot (separate terminal)
python bot.py
```

API docs available at `http://127.0.0.1:8000/api/docs/` (Swagger).

## API Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `POST` | `/api/check-user/` | No | Check if Telegram user is registered |
| `POST` | `/api/register/` | No | Register with `tg_login` + `password` → returns token |
| `POST` | `/api/login/` | No | Login with `identifier` + `password` → returns token |
| `POST` | `/api/messages/` | **Yes** | Send `user_text` → returns GPT-5.5 response |
| `GET` | `/api/messages/` | **Yes** | List message history for authenticated user |

**Auth header:** `Authorization: Token <key>`

**Request / response example:**

```bash
curl -X POST http://127.0.0.1:8000/api/messages/ \
  -H "Authorization: Token abc123" \
  -H "Content-Type: application/json" \
  -d '{"user_text": "What is the weather in Tokyo?"}'
```

```json
{"ok": true, "output": "The current temperature in Tokyo is 22°C."}
```
